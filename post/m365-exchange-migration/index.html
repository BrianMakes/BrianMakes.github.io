Study Guide - Migrating Email to Microsoft 365

When I was studying to complete my MCSA: Office 365 in 2018, I struggled a bit keeping the multiple migration options straight. I needed to work through the material again anyway, maybe my notes will help others that are studying.

I worked through the Exchange Online Docs and made a few notes.

Migration Types

Most of the study material for MS-100 talks about four primary migration methods: Staged, Cutover, Hybrid and IMAP. Lets walk through the decision making process together. We will take a look at the common question you need to ask yourself along the way.

###Exchange The first question we need to ask, “Are we migrating from Exchange?” If the answer to that question is yes, then we have three options to consider.

Cutover Migration

The Cutover Migration is supported when you are moving from Exchange 2003, Exchange 2007, Exchange 2010, or Exchange 2013. However, Staged Migrations should only be performed if there are fewer than 2000 mailboxes. Depending on the performance of your current Exchange server and bandwidth limits, you may see issues with migrations over 150 mailboxes.

Cutover Migrations are indicated when the source server is Exchange, and the number of mailboxes is less that 2000.

Staged Migration

The Staged Migration is supported when you are moving from Exchange 2003 and 2007 server, that have over 2000 mailboxes. Additionally, if you have an Exchange 2003 or 2007 migration that is being impacted by performance or bandwidth issues. There may be time when you will want to use Staged Migrations for 150 to 2000 mailbox migrations.

Staged Migrations are the best answer for large migrations (2000+ mailboxes) from Exchange 2003 and Exchange 2007.

Hybrid Migration

The Hybrid Migration is used for migrations over 150 (or 2000) for all version of Exchange since 2010. Microsoft identifies three different types of Hybrid migrations - full, minimal, and express.

  • Full - Large sized orgs, thousands of mailboxes, full integration back to on prem services
  • Minimal - medium sized orgs, few hundred to couple thousand mailboxes, limited integration
  • Express - small sized org, quick migration time frame

Hybrid Migrations are going to be the answer if you are dealing with more than 150 (2000) mailboxes on Exchange 2010 and on.

###Other Systems

IMAP Migration

IMAP Migration is the choice when you are migrating from email services like Gmail, Outlook.com, and other email systems that support the IMAP protocol.

When using IMAP for migrations, keep in mind that you will only be able to import mailbox folders. Things like tasks, calendar entries, and tasks cannot be migrated using IMAP.

###PST Import Worth noting on its own. If your organization has a large number of mailboxes exported to PST, you can use the Import Service to move the mail into Microsoft 365.

##Example Deployment Decisions Using the flow chart, and our previous notes, its going to be fairly easy to understand which migration method to use. Lets run through a couple of examples.

###Exchange 2007, low bandwidth We have been tasked with migrating from Exchange 2007 with 1500 users, to Microsoft 365.

  • Migrating from Exchange? - Yes
  • 2000 or Less Users? - Yes
  • 150 or Less Users? - No
  • Performance or Bandwidth issues? Yes Scenario identified bandwidth problems
  • Exchange 2003 or 2007 - Yes

The best choice for this scenario is Staged Migration.

###3rd Party Email Provider We have been tasked with moving 300 from Outlook.com to Microsoft 365.

  • Migrating from Exchange? - No
  • Does the current email provider support IMAP? Yes

The best choice for this scenario is IMAP Migration.

###Exchange 2013, 10,000 Mailboxes We have been tasked with migrating from Exchange 2013 with 10,000 users.

  • Migrating from Exchange? - Yes
  • 2000 or Less Users? - No
  • Exchange 2003 or 2007 - No

The best choice for this scenario is Hybrid Migration.

##Conclusion

If you are having trouble remember the details for each migration type during your exam prep, I recommend taking 15 minutes and drawing out a decision workflow for yourself. Active note taking has been shown to help improve comprehension and retention. There are only so many reasonable iterations of migration scenarios, take some time and run through the decision process to help prepare yourself to answer questions like these on the exam.
